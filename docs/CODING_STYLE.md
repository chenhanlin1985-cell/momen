# GDScript 代码规范

## 基本原则
- 一致性优先
- 可读性优先
- 扩展性优先
- 函数短小优先
- 静态类型优先

## 命名
- 文件：snake_case.gd
- 类：PascalCase
- 变量/函数：snake_case
- 常量：UPPER_SNAKE_CASE
- 事件/行动/tag/id：snake_case

## 代码顺序
1. class_name
2. extends
3. signal
4. enum
5. const
6. @export
7. 成员变量
8. 生命周期函数
9. 公共方法
10. 私有方法

## 类型
- 公共方法必须写参数和返回类型
- 集合尽量写清元素类型
- 能静态类型就不用弱类型

## 函数
- 单函数尽量不超过 40~60 行
- 一层函数只做一件事
- 复杂分支尽量拆成私有辅助函数

## 注释
- 注释解释“为什么”
- 不注释显而易见的“做了什么”
- 公共接口写清用途、输入、输出、副作用

## 禁止
- 巨型脚本
- God Object
- UI 里写核心规则
- 跨层直接改状态
- 硬编码节点路径驱动业务逻辑
