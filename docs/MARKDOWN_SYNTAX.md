# Markdown Syntax

## 定位
Markdown 是剧情作者的主配置文件。

运行时真正使用的仍然是这三层：
- `content/story/act1/csv/*`
- `content/dialogue/encounters/*`
- `content/dialogue/texts/*`

作者层只改 `content/story/act1/md/active/` 下面的小 md 文件，写完后统一编译回主结构。

## 作者层规则
作者层只允许出现两种可见标识：
- 数字编号
- 中文说明

作者层不应该出现：
- 英文运行时 ID
- 英文 NPC 缩写目录
- 英文文件名前缀
- 需要靠猜的隐式映射

## 目录规则
- 活跃剧情：`content/story/act1/md/active/`
- 归档和参考：`content/story/act1/md/archive/`
- `archive/` 不参与编译

活跃目录统一用数字文件夹：
- `00/`：通用事件
- `01/`：柳飞霞
- `02/`：药房执事
- `03/`：巡夜弟子
- `04/`：外门师兄
- `05/`：疯长老
- `06/`：王麻子

目录说明表：
- `content/story/act1/md/active/_folder_map.json`

这张表只保留数字目录和中文说明，不保留英文内部名。

## 文件命名
文件名统一用纯数字：
- `2001.md`
- `2002.md`
- `2003.md`
- `2004.md`
- `2005.md`
- `2007.md`
- `2008.md`
- `2102.md`

一个 md 只放一段小剧情。

## 头部字段
当前推荐只写：
- `# [事件] ...`
- `@编号: 2001`
- `@剧情分类: ...`
- `@内容分类: ...`
- `@表现形式: ...`
- `@触发阶段: ...`
- `@触发时间: ...`
- `@地点: ...`
- `@条件: ...`

不再要求作者手写 `@运行时ID`。
也不再要求作者手写 `@NPC` 或 `@参与者`。
人物归属由所在数字目录自动推断。

## 最小写法
```md
# [事件] 柳飞霞的试探
@编号: 2001
@剧情分类: 条件剧情事件
@表现形式: 对话事件
@触发阶段: 行动后
@触发时间: 白天
@地点: 茅草屋

[开场]
柳飞霞站在灵前，手里捏着一瓶金疮药，像是来吊唁，也像是来确认什么。

[观察]
她眼眶发红，但余光始终盯着你的储物袋。她真正关心的不是死者，而是遗物。

[入侵: 嗔]
你把“王麻子准备灭口”的念头轻轻埋进她心里。

[入侵提示: 嗔]
她的镇定裂开了一线，像是突然想起还有另一个人比你更危险。

[入侵记录: 嗔]
你让疑惧和怨恨一起在她心里翻了上来。

[对话]
=> [顺着她的话继续试探]
你没有立刻表态，只把话头往更深处引。
```

## 当前支持的段落
- `[开场]`
- `[观察]`
- `[入侵: 贪]`
- `[入侵: 嗔]`
- `[入侵: 痴]`
- `[入侵提示: 贪/嗔/痴]`
- `[入侵记录: 贪/嗔/痴]`
- `[对话]`
- `[对话: 贪/嗔/痴]`

兼容英文写法：
- `greed`
- `wrath`
- `delusion`

## 编号规则
- `@编号` 是作者层唯一主编号
- 编号必须是纯数字
- 编译器会用这个编号去匹配当前项目里的对应事件
- 如果项目里已经有这段剧情，编译器会写回原事件
- 如果项目里还没有这段剧情，编译器会用这个编号生成新事件

也就是说，作者只需要维护“编号是什么”，不需要维护“英文运行时 ID 是什么”。

## 编译后会写回什么

### 写回 `encounters / texts`
- `[开场]` -> `opening_text_id`
- `[观察]` -> `observation_text_id`
- `[入侵:*]` -> `desc`
- `[入侵提示:*]` -> `hint`
- `[入侵记录:*]` -> `log`
- `[对话:*]` -> `option_overrides`

### 写回 CSV
`=>` 下面的默认对话选项、结果、条件、效果会同步写回：
- `events.csv`
- `event_triggers.csv`
- `event_options.csv`
- `option_conditions.csv`
- `option_effects.csv`
- `localization.csv`

## 新增一个 NPC 流程怎么做
如果是“现有 NPC 增加一段后续流程”，按这 6 步：

1. 先查目录号。
   看 `content/story/act1/md/active/_folder_map.json`。

2. 在对应数字目录里新建一个纯数字文件名。
   例如柳飞霞新增一段，可在 `01/` 里建 `2201.md`。

3. 写头部。
   至少写：
   - `# [事件] ...`
   - `@编号: ...`
   - `@剧情分类: ...`
   - `@表现形式: 对话事件`
   - `@触发阶段: ...`
   - `@触发时间: ...`
   - `@地点: ...`

4. 写三阶段内容。
   先写 `[开场]`、`[观察]`，再写 `[入侵:*]`，最后写 `[对话]` 和需要的 `[对话:*]`。

5. 如果需要状态机推进，在结果里继续写条件和效果。

6. 写完后执行一次编译。

```powershell
C:\momen\Godot\Godot_v4.6-stable_mono_win64\Godot_v4.6-stable_mono_win64_console.exe --headless --editor --path C:\momen -s res://tools/story_compiler/compile_current_markdown_runner.gd --quit
```

## 什么时候还要补别的地方
只有在“新增的是一个全新 NPC”时，底层系统里才通常需要再补这些：
- `content/story/act1/npc_name_map.json`
- `content/npcs/npc_definitions.json`
- `content/npcs/npc_interactions.json`
- 这个 NPC 对应的 `content/dialogue/encounters/*`
- 这个 NPC 对应的 `content/dialogue/texts/*`

如果只是给现有 NPC 追加一段流程，通常主入口就是新增 md 文件，然后编译。

## 额外规则
- 作者层不要手写选项英文 ID；`=> [选项文本]` 就够了。
- 编译器会根据剧情编号和选项顺序稳定生成底层选项编号。
- 作者层目录、文件名、头部编号都只用数字；说明都用中文。

## 当前支持的常用效果别名
作者层效果行继续写在 `$...$` 里，当前常用且已验证支持的中文别名包括：

- `气血`
- `暴露度`
- `神识`
- `线索碎片`
- `灵石`
- `污染度`
- `天魔经验`
- `获得状态`
- `移除状态`
- `NPC关系`
- `获得卡牌`
- `移除卡牌`
- `结束本局`

示例：

```md
$线索碎片: +1, 天魔经验: +1, 获得卡牌: 暗报折纸$
```

选项也支持在标题后括号里写资源消耗，例如：

```md
=> [花两块灵石换一张牌](消耗灵石: 2)
```
