# 事件展示统一标准

## 目标

当前项目已经形成了多种节点类型，但主界面上的展示方式仍然混杂了旧结构与新结构。  
本标准的目标不是先做包装，而是先保证玩家在任意节点里都能快速回答 3 个问题：

1. 我现在遇到的是什么类型的事件
2. 这个事件通常会带来什么
3. 我现在应该做什么

## 一级标准

所有主要节点面板都应尽量遵守同一信息顺序：

1. 类型标题  
   例如：`随机事件`、`奖励事件`、`回顾事件`、`商店事件`、`对话战斗事件·精英战`
2. 类型说明  
   用一句话说明这一类事件的主要作用与风险/收益
3. 当前主体  
   当前真正发生的内容，如人物、标题、正文、战斗对象
4. 当前决策  
   玩家这一拍可以执行的选项
5. 次要提示  
   检定提示、阶段提示、拖拽提示、翻页提示、状态说明

## 二级标准

### 普通事件面板

- 顶部必须先显示事件类型
- 类型下方必须有一句说明
- 正文标题与正文内容属于“当前主体”
- 选项区域统一命名为 `当前决策`

### 对话面板

- 不再只显示“人物对话”这种泛标签
- 顶部类型必须反映真实节点类型，例如奖励型对话、剧情对话、对话战斗入口
- 人物名、人物身份、阶段标签属于“当前主体”
- 选项区域统一视作 `当前决策`
- `观察 / 入侵` 等按钮属于决策，不属于标题信息

### 战斗面板

- 顶部标题必须先说明这是普通战 / 精英战 / BOSS 战
- 第二行必须直接说明这场战斗的意义
- 敌人立绘、心防、计算 HUD 属于“当前主体”
- `重抽 / 植入魔念` 属于“当前决策`
- 抽牌、弃牌、拖拽提示属于次要提示

## 三个禁止项

- 不要把关键事件信息只放在小标签里
- 不要让玩家通过颜色猜事件类型，文字必须先讲清楚
- 不要让“当前决策”区域有时有标题、有时完全没有标题

## 当前执行策略

本轮先做信息统一，不先做视觉包装统一：

- 统一类型标题
- 统一类型说明
- 统一 `当前决策` 区域命名
- 统一各类面板的信息顺序

等这层稳定后，再做不同事件类型的视觉包装强化。
