# 低信息量剧情节点压缩方案

## 问题判断

当前界面“不整齐”的根因，不只是不同面板长得不一样，而是：

- 太多桥接节点仍然占用完整事件面板
- 有些节点只有一句说明和一个按钮，却走了完整的“类型 -> 说明 -> 主体 -> 提示 -> 决策”流程
- 玩家频繁被迫停下来读一拍“作者解释为什么下一拍要发生”，而不是在做选择或处理结果

这会让中段体验变成：

- 事件很多
- 交互很少
- 信息密度低
- 节奏被切碎

所以后续不应该再单纯追求“所有面板结构一样”，而应该先把事件按**信息密度和操作价值**拆开。

## 新的表现层级

后续建议不再只有一种“完整事件面板”，而是至少分成 3 层。

### 1. 完整节点

适用场景：

- 有真正决策
- 有明显风险 / 收益
- 有战斗
- 有分支或检定

保留完整大面板。

适合保留完整节点的现有类型：

- `2001 / 2004 / 2005 / 2003`
- `9101 / 9201 / 9301 / 9401 / 9501`
- `3301 / 3302 / 3401 / 3402`
- 有明确多选、检定、买卖、删牌、恢复的随机 / 奖励 / 商店 / 回顾事件

### 2. 紧凑节点

适用场景：

- 主要职责是桥接路线
- 信息量中等
- 只有 1 到 3 个高概括选项
- 不值得占满一个完整大面板

表现建议：

- 保留类型标题
- 保留 1 到 2 句摘要
- 选项直接紧凑呈现
- 不再铺很长正文和大片留白

这类最适合当前项目。

### 3. 摘要节点

适用场景：

- 只有单句结果
- 只有一个继续按钮
- 本质上是“系统反馈”而不是“事件”

表现建议：

- 不单独占一张完整事件卡
- 直接做成顶部提示、日志摘要或过场条
- 自动或半自动推进

## 推荐的游戏化改法

## A. 把桥接主线改成“路线抉择卡”

例如 `1101-1104`：

- 不做成长段正文
- 做成一句局势总结
- 下方直接给 2 到 4 个路线按钮
- 每个按钮写清楚：
  - 你在追什么
  - 偏风险还是偏收益
  - 会更靠近哪条人物线

这比“先读一大段桥接，再看按钮”更像游戏。

## B. 把持续推进节点改成“线索推进卡”

例如 `2301-2303 / 2401-2403 / 1301-1303 / 2201-2203`：

- 顶部只保留一句当前新判断
- 正文压缩成 1 到 2 句结果描述
- 选项不再像小说段落，而像推进动作

例如：

- `继续往旧账里挖`
- `先压住风声`
- `接下这份投名状`
- `把异常当退路`

玩家需要的是“我现在推动哪条线”，不是“作者再次解释这条线是什么意思”。

## C. 把战后收束改成“战斗结算条”

例如 `9102 / 9202 / 9302 / 9402 / 9502`：

- 显示胜负
- 显示经验 / 卡牌 / 状态变化
- 用 1 句人物后果收束
- 然后直接继续

这些节点更适合：

- 战斗结果标题
- 奖励摘要
- 一句后果
- 一个 `继续`

而不是再展开成完整事件页。

## 建议新增的表现类型

当前只有：

- `standard_event`
- `dialogue_event`
- `battle_event`
- `ending_event`

建议后续新增：

- `compact_choice_event`
  用于路线桥接与二层推进
- `summary_event`
  用于战后收束与单句结果

## 具体落地顺序

### 第一步

先不动内容文本，只改表现层：

- `1101-1104`
- `2201-2203`
- `2301-2303`
- `2401-2403`
- `1301-1303`

这批统一切到 `compact_choice_event`

### 第二步

把：

- `9102 / 9103`
- `9202 / 9203`
- `9302 / 9303`
- `9402 / 9403`
- `9502 / 9503`

统一切到 `summary_event`

### 第三步

再回头压文本，把桥接文案改成：

- 1 句局势总结
- 1 句路线含义
- 直接给决策

## 结论

后续目标不应该是“所有事件都长得更一致”，而应该是：

- 高价值节点保留完整面板
- 桥接节点压成紧凑节点
- 结果节点压成摘要节点

这样才会更游戏化，也能真正解决“桥接过多、繁琐、信息量低”的问题。
