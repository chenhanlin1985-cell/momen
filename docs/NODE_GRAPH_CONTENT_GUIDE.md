# 节点链路图内容规范

## 定位
当前项目的内容组织方式，已经不再是“地点探索 + 自由行动”。

更适合把整局游戏理解为一张节点链路图：
- 有确定起点
- 有确定终点
- 中间允许分叉、回收、条件分流、随机扰动
- 玩家每次真正做的，是在当前节点上推进到下一个节点

这份文档只解决两件事：
1. 后续内容应该按什么单位来补
2. Act 1 现有骨架上，下一批节点应该往哪里补

## 单位
后续补内容时，最小生产单位不是“地点”，也不是“NPC 可点击项”，而是“节点”。

一个节点至少包含这 6 个部分：
- 编号：纯数字
- 进入条件：什么时候能进来
- 节点正文：这一拍实际发生什么
- 可选分支：玩家能怎么回应
- 结果变化：会改哪些状态、标签、关系、资源
- 去向：会把后续内容推向哪一类节点

在当前项目里，一个节点通常对应：
- 一个 `events.csv` 事件
- 一组 `localization.csv` 文本
- 若是对话节点，再对应一份 `md/active/*.md`

也就是说，后面“补内容”，本质上就是补节点。

## 三类节点
项目当前运行时已经稳定成三类节点，这三类不要再混。

### 1. 主线节点
用途：
- 定节奏
- 定阶段目标
- 把一天或一章往下推

当前对应：
- `fixed_story`
- `ending_check`

特点：
- 必然发生
- 是链路图主骨架
- 不负责提供大量细节，而是负责收束前面已经积累的状态

### 2. 条件节点
用途：
- 回应玩家之前做过的事
- 让某条人物线或某条调查线真正成立

当前对应：
- `conditional_story`

特点：
- 不是每轮都触发
- 是“你之前留下的状态开始回响”
- 用来把柳飞霞、药房执事、巡夜弟子、疯长老这些线真正织进主骨架

### 3. 随机节点
用途：
- 补密度
- 补氛围
- 补风险和资源波动

当前对应：
- `random_filler`

特点：
- 不能抢主线位置
- 不能承担关键设定揭示
- 只能增强“这一轮和上一轮不完全一样”

## 当前项目中的映射关系
为了不增加新技术债，现阶段先不新增第二套“链路图专用语法”。

先按当前真实运行结构来落：
- 主线节点：`content/story/act1/csv/events.csv`
- 条件节点：`content/story/act1/csv/events.csv`
- 随机节点：`content/story/act1/csv/events.csv`
- 对话正文作者入口：`content/story/act1/md/active/`
- 对话运行时写回：`csv + encounters + texts`

这意味着：
- 设计层可以按“链路图”思考
- 作者层仍然按现有 Markdown 规范写
- 运行层仍然按现有 CSV / encounter / texts 结构执行

不要现在就为了“链路图”再发明一套新运行格式。

## 编写规则
后面补节点时，统一按下面 8 条执行：

1. 一个编号只表达一个节点，不在同一编号里塞两段不同用途的内容。
2. 主线节点只负责“推进阶段”，不要在主线节点里堆太多枝节说明。
3. 关键设定揭示优先放条件节点，不放随机节点。
4. 随机节点只补密度、氛围、资源与风险，不承担不可替代的信息。
5. 每个新节点都要回答“它接在谁后面、它改变什么、它会把谁推出来”。
6. 所有编号继续用纯数字，不回到英文命名。
7. 玩家可见文本继续全部外置，不回流进脚本。
8. 不为了表现“链路图”去额外加兼容层；先补内容，再视需要补工具。

## 节点设计模板
后续设计新节点时，先用下面这个模板思考，再落到具体 md / csv。

```text
节点编号：
节点类型：主线 / 条件 / 随机
所属人物或主题：
进入前提：
节点目的：
节点正文核心：
玩家可选分支：
结果变化：
后续去向：
是否必须可重复：
```

## Act 1 最值得补的节点空位
下一阶段最值得补的，不是再加新系统，而是补下面 5 类空位。

### 1. 主线之间的桥接节点
现在 `1001` 到 `1006` 是骨架，但中间很多过渡还薄。

最该补的是：
- 从“族兄之死”到“必须调查”的桥接节点
- 从“柳飞霞试探”到“执事压迫”的桥接节点

用途：

- 不抢人物线
- 负责把大节点之间连顺

### 2. 柳飞霞线的第二层分叉节点
柳飞霞现在已经有起手和延伸，但“信她、利用她、反试她”这三条味道还不够分明。

最该补的是：
- 你主动顺着她演
- 你拿族兄遗物试她
- 你让她误以为王麻子要先灭她口

建议编号段：
- `2201-2299`

用途：
- 让柳飞霞真正成为可经营、可反制、可误导的一条线

### 3. 王麻子线的持续压迫节点
现在这条线有压力感，但还缺“他为什么越来越危险”的持续节点。

最该补的是：
- 小额勒索
- 记账、盯梢、放风
- 借手下立规矩
- 月考前的卡脖子

建议编号段：
- `2401-2499`

用途：
- 让“外门秩序的压迫感”不是一句设定，而是不断回来的节点

### 4. 疯长老线的生死交易节点
疯长老现在更像阶段关卡，但还缺“危险机遇线”的层次。

最该补的是：
- 试药前的征兆
- 试药后的代价
- 以秘密换庇护
- 以资源换活命技术

建议编号段：
- `2301-2399`

用途：
- 把他从单点生死门，扩成一条高风险高回报的路线

### 5. 化骨池异常线的回响节点
这条线现在有氛围，但还缺“异常越来越贴近你”的连续感。

最该补的是：
- 白天残痕
- 夜间异声
- 错名呼唤
- 污染后的反应

建议编号段：
- `1301-1399`

用途：
- 让天魔身份感真正进入日常推进
- 让异常不是单独一晚的插曲

## 补内容的优先级
如果只按“让 Act 1 立刻更丰满”来排，建议顺序如下：

1. 先补主线桥接节点
2. 再补柳飞霞第二层分叉
3. 再补王麻子持续压迫
4. 再补疯长老高风险交易
5. 最后补化骨池异常回响

原因很简单：
- 先补桥接，整章才会顺
- 再补人物分叉，玩家才会觉得选择有重量
- 最后补异常和氛围，整章才会“活起来”

## 对 Markdown 作者层的要求
在作者层，不要把自己当成“写散文剧情”，而要把自己当成“给链路图补节点”。

每写一个新 md，先回答这 4 个问题：
- 这个节点接在哪个已存在节点后面
- 它属于哪条线
- 它改了哪些 flag / tag / relation / resource
- 它是为了把哪一个后续节点推出来

如果回答不清，这个节点大概率就还不该写。

## 当前结论
后续补内容时，最重要的不是再扩系统，而是坚持下面这条主原则：

不是“给地点加内容”，而是“给链路图补节点”。

只要节点持续补对：
- 主线会更顺
- 支线会更厚
- 选择会更有重量
- 随机内容也会更有位置感

这才是当前项目最有效的增量方式。
