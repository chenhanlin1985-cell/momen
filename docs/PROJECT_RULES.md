## 13. 文本与代码隔离
- 代码文件里不允许直接写剧情正文、按钮文案、长段说明、角色对白。
- GDScript、场景脚本、工具脚本里只允许保留结构性键名、状态键、资源路径、短的技术性提示。
- 所有面向作者和玩家的可读文本，统一放在 Markdown、CSV、本地化表、texts JSON 等文本资源里。
- 如果某段文本未来可能被改文案，就不允许写死在代码里，必须外置。
- 这条规则的目的不是“风格统一”，而是避免编码问题污染代码、场景和编译器，降低解析错误和乱码风险。

## 14. 资源目录规则
- 所有美术、音频、字体、图标等资源统一放在 `res://assets/`。
- `res://content/` 只保留配置、文本、CSV、Markdown、JSON 等内容数据，不再存放图片、音频、字体资源。
- NPC 立绘统一放在 `res://assets/art/portraits/npcs/`。
- `npcs/` 下的子目录只允许使用数字编号，不允许直接使用带含义的英文目录名。
- 如果需要表达目录含义，统一通过对照表文件说明，不把含义直接写进目录名。
- 资源文件名也优先使用数字编号或数字前缀，不直接暴露带含义的英文命名。
- 配置文件中的资源路径统一写成 `res://assets/...`。
- 新增资源时，先放 `assets/`，再在 `content/` 的配置文件里引用，不要反过来。

## 15. 编码与转码规则
- 项目中的 `.gd`、`.tscn`、`.tres`、`.json`、`.csv`、`.md` 统一使用 `UTF-8` 编码。
- 默认要求使用 `UTF-8 without BOM`，除非 Godot 某类资源文件明确要求其他格式。
- 不允许在编辑过程中反复切换编码，不允许把同一个文件在 GBK、ANSI、UTF-8 之间来回转码。
- 如果某个文件已经出现乱码，优先回到可信源内容重新写入，不允许继续在乱码文件上做二次转码“修复”。
- 命令行写文件时，不允许使用依赖系统默认编码的方式直接覆盖文本文件。
- 如果必须通过脚本写文件，必须显式指定 `UTF-8`，并明确是否带 `BOM`。
- 优先使用不会隐式改编码的编辑方式；修改文本资源时，必须先确认原文件编码，再保持一致写回。
- 新增项目规则、剧情文本、说明文档时，不允许把中文直接写进不受控的默认转码命令里。
